# XJTU-SY Bearing Dataset 解析 - 条件変動環境での異常検知の限界と可能性

製造業15年（品質管理10年）のエンジニアが、予知保全プロダクト開発の過程で取り組んだ解析です。
CWRUデータ（実験室・綺麗すぎるデータ）→ NASAデータ（自然劣化・条件固定）に続く、
**難易度★★★の条件変動データセット**です。

---

## なぜXJTU-SYを選んだか

NASAベアリングデータセットは「条件固定・自然劣化」という現実に近いデータでした。
ただし条件が固定されているという前提が、現場とは違う。

実際の工場では、

```
同じ設備でも
→ 加工する素材が変わる
→ 送り速度が変わる
→ 荷重が変わる
→ 「正常の振動パターン」が毎回違う
```

XJTU-SYは3種類の運転条件で15本のベアリングを破損まで走らせたデータで、
「条件が変わると予知保全はどう難しくなるか」を検証できる唯一のデータセットです。

---

## データ概要

- **15本**のベアリングをrun-to-failure（破損まで連続運転）
- **3条件 × 5本**の構成：

| 条件 | 回転数 | 荷重 | 特徴 |
|------|--------|------|------|
| Condition1 | 2100rpm | 12kN | 低速・高荷重 |
| Condition2 | 2250rpm | 11kN | 中速・中荷重 |
| Condition3 | 2400rpm | 10kN | 高速・低荷重 |

- サンプリング：25.6kHz、1分ごとに1.28秒収録
- 水平・垂直の2チャンネル
- 損傷モード：内輪摩耗・外輪摩耗・外輪破損・ケージ破損など多様

## 解析で見えてきたこと（現場目線のコメント）

### 1. NASAで発見したパターンが条件変動でも再現した

前回のNASA解析で発見した「ピーク→フェードアウト→破損」パターンを覚えているだろうか。

```
正常期    → 振動が安定
異常予兆期 → 振動が増加し始める
一時安定期 → 振動がいったん落ち着く  ← 「調子が戻った」ではなく「さらに壊れた」
急激劣化期 → 振動が急上昇して破損
```

Condition1の5本中4本（Bearing1_1, 1_3, 1_4, 1_5）でこのパターンを確認。
条件が変動する複雑な環境でも同じパターンが現れたことで、
**このパターンはベアリング劣化の普遍的な現象である可能性が高い。**

1本（Bearing1_2）は連続劣化パターンだった。損傷モードの違いによる可能性がある。

**現場経験との照合：**
「最近機械の調子がよくなってきた気がする」という言葉が出る時期が危ない。
このパターンを知っていれば、その言葉が出た直後に緊急点検を入れる判断ができる。

---

### 2. 条件によって「正常の基準」が1.5倍違う

| 条件 | 正常時RMS平均 | Cond1比 |
|------|-------------|---------|
| Condition1 (2100rpm/12kN) | 0.623 | 1.00倍 |
| Condition2 (2250rpm/11kN) | 0.407 | 0.65倍 |
| Condition3 (2400rpm/10kN) | 0.486 | 0.78倍 |

**正常時のRMSが条件によって最大1.5倍違う。**

これは「固定閾値では使えない」ことを意味する。

**現場経験との照合：**
職場の予知保全システムが1800rpm固定で運用されている理由がここにある。
回転数を固定しないと「正常の基準」がずれて誤報が出やすくなる。
現場の先人たちが経験的に知っていたことを、データが証明した形だ。

---

### 3. 条件が変わると過検出が爆発する（最重要の発見）

この表の数字は**異常検出率（全スナップショットのうち「異常」と判定された割合）**であり、精度ではない点に注意。

| 条件 | Cond1学習モデルの異常検出率 | 自己学習モデルの異常検出率 | 差分 |
|------|--------------------------|--------------------------|------|
| Condition1 | 44.7% | 44.7% | ±0% |
| Condition2 | **96.7%** | 49.5% | +47.3% |
| Condition3 | **90.8%** | 12.7% | +78.1% |

Condition2を例にすると、491スナップショット中475個（96.7%）を異常と判定している。
自己学習モデルでは243個（49.5%）なので、残りの232個は**「正常なのに異常と誤判定」**している。

何が起きているかというと、

```
Condition1の正常時RMS = 0.623
Condition2の正常時RMS = 0.407

→ ベースラインが1.5倍違う
→ Cond1モデルから見ると
→ Cond2の正常状態が「いつもより静かすぎる = おかしい」に見える
→ 正常区間のデータまで全部「異常」と判定してしまう
```

**この表で示せていること・示せていないこと**

- 条件が変わると過検出（False Positive大量発生）が起きる
- 「劣化の検知精度が落ちる」とは言えない（正解ラベルがない教師なし学習のため）

本来は「正常区間の誤報率」と「末期区間の検出率」を分けて評価すべきで、それは今後の課題。

**これは現場で実際に起きる最悪のシナリオで、**

誤報が続くと現場の人間が「このシステム使えない」と判断してシステムを止める。
予知保全の導入失敗事例の多くがこのパターンだと思われる。

**現場への示唆：**
回転数や荷重が変わるたびに、モデルの再学習または正規化が必須。
「設備の条件変更」はソフトウェアのバージョンアップと同じくらい重要なイベントとして扱うべきだ。

## CWRUデータセット・NASAデータセットとの比較

| 項目 | CWRU | NASA/IMS | XJTU-SY |
|------|------|----------|---------|
| 条件 | 固定・実験室 | 固定・実稼働 | 変動・3種類 |
| 劣化パターン | 明確・単純 | 自然劣化 | 自然劣化・多様 |
| 転移学習の難しさ | 低 | 中 | 高 |
| 現場への近さ | 遠い | 中程度 | 近い |
| 教師あり精度 | 100% | 高精度 | 条件依存で大きくばらつく |

**3つのデータセットを通じた結論：**

データが綺麗になるほど精度は高くなるが現場から遠ざかる。
XJTU-SYで初めて「現場の難しさ」に近い問題に直面した。

## この問題の解決方向性

条件変動に対応するための技術的アプローチ：

**① 条件正規化**
回転数・荷重に応じてデータを正規化してから異常検知する。
「この条件での正常パターン」を条件ごとに学習させる。

**② 転移学習・ドメイン適応**
異なる条件間の共通パターンを抽出して汎用モデルを作る。
次のFEMTO/PRONOSTIAデータセットで検証予定。

**③ 条件をモデルの入力に含める**
回転数・荷重を特徴量として入れて「この条件ではこのRMSが正常」を学習させる。
現場では最もシンプルで実用的なアプローチ。
