# NASA Bearing Dataset 解析 - 劣化過程の可視化と異常検知

製造業15年（品質管理10年）のエンジニアが、予知保全プロダクト開発の過程で取り組んだ解析です。
単なるモデル構築ではなく、「現場で何が起きているか」を理解することを目的としています。

---

## なぜこのデータセットを選んだか

CWRUデータセット（Case Western Reserve University）は実験室で意図的に損傷を作った「綺麗すぎるデータ」です。
正常と損傷の差が大きすぎて、RMSが最大9倍も違う。これは現場で本当に検知したい「壊れかけ」のデータではない。

NASAベアリングデータセットは違います。
実際の設備が**自然に劣化して破損するまでの約7日間**を記録したデータです。
現場エンジニアが「予知保全で何を検知したいか」に一番近い。

---

## データ概要

- **テスト2** を使用（最も劣化が明確に進行）
- **984スナップショット**（約7日間・1時間ごとに1分間収録）
- **4チャンネル**の振動センサー
- **20kHz**サンプリング

## 解析で見えてきたこと（現場目線のコメント）

### 1. RMSの急上昇は故障直前に集中する

スナップショット#900あたりからRMSが急上昇します。
7日間のほとんどは「正常に見える」状態で、最後の数時間で一気に変化する。

**現場経験と照らし合わせると：**
これは工作機械の主軸ベアリングが壊れる直前の感覚と一致します。
「なんかおかしい」と感じてから、実際に加工精度が落ちるまでの時間が短い。
だから「定期点検」では間に合わないことがある。

### 2. 「ピーク→フェードアウト→破損」という意外なパターン

解析の中で一番驚いたのがこの現象です。
劣化が進む途中で一度振動が**落ち着く時期**があります。

```
正常期（#1〜#500）     → 振動が安定・低い
異常予兆期（#500〜#800）→ 振動が増加し始める
一時安定期（#800〜#850）→ 振動がいったん落ち着く  ← ここが危ない
急激劣化期（#850〜#984）→ 振動が急上昇して破損
```

**この「一時安定期」の正体：**
損傷した部分が削れて、一時的に「引っかかり」がなくなる現象です。
つまり「振動が落ち着いた」＝「回復した」ではなく、「さらに壊れた」サインです。

品質管理の現場で「最近調子よくなった気がする」と言われる機械が、直後に壊れる経験と重なります。
このパターンを知っていれば、「一時安定期」こそ緊急メンテナンスのタイミングだと判断できます。

### 3. 尖度（Kurtosis）は末期に跳ね上がる

RMSが「平均的な振動の強さ」を示すのに対して、尖度は「衝撃的な振動の突発性」を示します。

- 正常時：Kurtosis ≈ 3（正規分布に近い）
- 末期：Kurtosis >> 10（衝撃的な振動が頻発）

**現場的な解釈：**
「ゴリゴリ」という連続音から「ガタッ」という衝撃音に変わる感覚です。
RMSだけを監視していると末期サインを見逃す可能性があります。
RMS + 尖度の組み合わせで監視するのが現実的と考えています。

### 4. 異常検出率47.7%は「多い」のか「少ない」のか

解析結果では全スナップショットの約半分が異常判定されました。
最初は「感度が高すぎるのでは」と思いましたが、分布を見ると後半に集中しています。

これは正しい挙動で、7日間のデータのうち後半3〜4日が「劣化期間」だったことを意味します。
閾値の設定を厳しくすれば誤報を減らせますが、早期検知を犠牲にします。
**「いつ知らせるか」はデータではなく、現場の運用設計で決める問題です。**

## CWRUデータセットとの比較

| 項目 | CWRU | NASA |
|------|------|------|
| データの性質 | 実験室・意図的損傷 | 実稼働・自然劣化 |
| 正常と異常の差 | 大きい（RMS9倍差） | 小さく徐々に変化 |
| 教師あり精度 | 100% | 現実的な難しさあり |
| 現場への近さ | 遠い | 近い |
| 使いどころ | モデルの基礎構築 | 劣化過程の理解 |

**両方を知ったうえでの結論：**
CWRUで「どんな特徴量が重要か」を学んで、NASAで「実際の劣化とはどういうものか」を理解する。
この2つを組み合わせることで、実機への適用に近いモデルが設計できると考えています。
