# FEMTO/PRONOSTIA Bearing Dataset 解析 - 複合損傷環境での異常検知の現実

製造業15年（加工技術5年・品質管理10年）のエンジニアが、予知保全プロダクト開発の過程で取り組んだ解析です。
CWRUデータ（実験室・綺麗）→ NASAデータ（自然劣化・条件固定）→ XJTU-SY（自然劣化・条件変動）に続く、
**シリーズ最終章・複合損傷データセット**です。

---

## なぜFEMTOを選んだか

XJTU-SYで「条件が変わると正常ベースラインが変わり過検出が爆発する」という問題を発見した。
しかし現場にはもう一つ別の難しさがある。

```
同じ設備・同じ条件でも
→ 内輪から壊れるベアリング
→ 外輪から壊れるベアリング
→ ケージが先に折れるベアリング
→ 「何が壊れているかわからない」
```

FEMTOは損傷を最初から付けず自然劣化させているため、複数箇所が複合的に損傷する。
これが「AI導入したのに現場で使えなかった」という失敗の根本原因に最も近いデータセットだ。

---

## データ概要

- **17本**のベアリングをrun-to-failure（破損まで連続運転）
- **3条件 × 訓練・テスト分割**：
  - Condition1：1800rpm / 4kN
  - Condition2：1650rpm / 4.2kN
  - Condition3：1500rpm / 5kN
- サンプリング：25.6kHz、10秒ごとに0.1秒収録
- 水平・垂直の2チャンネル（振動）＋温度センサー
- **損傷モード：内輪・外輪・ボール・ケージが複合的に発生**

---

## 解析で見えてきたこと（現場目線のコメント）

### 1. 「ピーク→フェードアウト→破損」パターンが3データセットで再現

NASAで発見し、XJTU-SYで条件変動環境でも確認したパターンが、複合損傷環境でも6本中5本で現れた。

```
正常期    → 振動が安定
異常予兆期 → 振動が増加し始める
一時安定期 → 振動がいったん落ち着く  ← 「さらに壊れた」サイン
急激劣化期 → 振動が急上昇して破損
```

**3データセット・異なる条件・異なる損傷モードを通じて同じパターンが現れた。**
これはベアリング劣化の普遍的な現象と言ってよい。

現場でこのパターンを知っているかどうかで、「最近調子が戻ってきた」という言葉への対応が180度変わる。

---

### 2. 同じ条件でも損傷モードで劣化の形が全く違う（最重要の発見）

| | Bearing1_1 | Bearing1_2 |
|---|---|---|
| 寿命 | 2803スナップショット | 871スナップショット |
| 劣化開始 | 寿命の**0.9%**地点 | 寿命の**94.8%**地点 |
| RMS最大/正常比 | 16.2倍 | 6.0倍 |
| 劣化パターン | 早期から徐々に劣化 | ほぼ最後まで正常→一気に壊れる |

**同じ設備・同じ条件なのに、劣化の始まるタイミングが全く違う。**

Bearing1_1は寿命の1%地点から徐々に劣化が始まり、長い時間をかけて破損する。
Bearing1_2はほぼ最後まで正常に見えて、突然一気に壊れる。

**現場でのAI導入失敗との直接的な繋がり：**

```
「AIを導入したのに突然壊れた」
→ Bearing1_2タイプのベアリングだった可能性が高い
→ AIが「異常なし」と言い続けていた
→ 突然壊れた
→ 「AIは使えない」という結論になる
```

しかし正確には「このタイプのベアリングはこういう壊れ方をする」という事実を知らずに導入したのが問題だ。
AIが悪いのではなく、運用設計が間違っていた。

---

### 3. 末期検出率に大きなばらつきがある

この表の数字は**異常検出率（全スナップショットのうち「異常」と判定された割合）**であり、精度ではない。
XJTU-SYの教訓を踏まえ、正常区間と末期区間を分けて評価した。

| Bearing | 誤報率（正常区間20%） | 末期検出率（最後10%） |
|---------|-------------------|-------------------|
| Bearing1_1 | 5.0% | **100.0%** |
| Bearing1_2 | 5.2% | **52.9%** |
| Bearing2_1 | 5.5% | 70.3% |
| Bearing2_2 | 5.0% | **100.0%** |
| Bearing3_1 | 5.8% | 70.6% |
| Bearing3_2 | 5.2% | 63.8% |

**誤報率は全ベアリングで約5%に安定している一方、末期検出率は53〜100%と大きくばらつく。**

Bearing1_2の末期検出率が52.9%というのは、「最後まで正常に見える」タイプだから。
IsolationForestが「正常」と学習したパターンからほとんど外れないまま壊れていく。

**示せていること：**
- 誤報率は条件に関わらず安定して低く抑えられる
- 末期検出率は損傷モードによって大きく変わる

**示せていないこと：**
- 「一気に壊れるタイプ」を事前に識別する方法
- 末期検出率を上げるための特徴量設計（今後の課題）

**4データセットを通じて得た設計思想：**


# 個体別学習 vs 共通モデル - 「突然壊れる個体」への挑戦

FEMTO/PRONOSTIAデータセットを使った追加検証です。
前回の解析で発見した「Bearing1_2問題」への新しいアプローチを試みました。

---

## なぜこの検証をしたか

前回の解析で見えた未解決の問題があります。

```
Bearing1_2
→ 寿命の94.8%地点まで正常に見える
→ 突然一気に壊れる
→ 末期検出率52.9%
→ 「AI導入したのに突然壊れた」の典型
```

この問題への仮説として浮かんだのが**個体別学習**です。

```
現状の共通モデル
→ 複数個体のデータを混ぜて学習
→ 「一般的な正常パターン」を覚える

個体別学習
→ 最初のN%だけで学習
→ 「この個体の正常パターン」を覚える
→ 個体差に対応できるのではないか
```

ベアリングの寿命設計はL10寿命（100個中90個が生き残る時間）という統計値が基準だが、
現場では「精度が悪くなったら交換」という経験則で運用されている。
個体差があるのは現場では常識で、それをデータで捉えようとした検証です。

---

## 検証設計

- 学習期間：各ベアリングの最初の**10%・20%・30%**のデータで学習
- 評価指標：
  - 正常区間の誤報率（最初20%）
  - 末期区間の検出率（最後10%）
- 比較：個体別モデル vs 共通モデル（全個体の最初N%を合わせて学習）

全ての数値は**異常検出率（異常判定割合）**であり精度ではない

---

## 結果

### 個体別モデルが優位なケース（4本）

| Bearing | 個体別 | 共通 | 差分 | 理由 |
|---------|--------|------|------|------|
| Bearing2_2 | 100.0% | 72.2% | **+27.8%** | 自分の正常を正確に学習→逸脱を検出 |
| Bearing3_2 | 66.3% | 29.4% | **+36.8%** | 同上 |
| Bearing3_1 | 62.7% | 49.0% | +13.7% | 同上 |
| Bearing2_1 | 65.9% | 68.1% | -2.2% | ほぼ同等 |

### 共通モデルが優位なケース（Bearing1_2）

| 学習期間 | 個体別 | 共通 | 差分 |
|---------|--------|------|------|
| 10% | 52.9% | **85.1%** | -32.2% |
| 20% | 52.9% | **62.1%** | -9.2% |
| 30% | 51.7% | 52.9% | -1.1% |

**学習期間10%の共通モデルが最もBearing1_2の末期を検出できた。**

---

## なぜBearing1_2では個体別学習が効かないか

```
個体別モデルの弱点
→ 「自分の正常パターン」を正確に学習してしまう
→ Bearing1_2は最後まで振動が小さいまま壊れる
→ 破損直前まで「自分の正常範囲内」と判定してしまう

共通モデルの意外な強み
→ 他の個体の正常パターンも含めて学習している
→ Bearing1_2の微妙な変化を「他と違う」として検出できる
→ 特に学習期間が短い（10%）ほど効果的
→ 少ないデータで「一般的な正常」を覚えているため
→ 僅かな逸脱にも敏感になる
```

---

## 誤報率の安定性

**個体別モデルの最大の強み：**
誤報率が全ベアリング・全学習期間で**約5%に安定**。

**共通モデルの弱点：**
学習期間30%で最大**20.1%**まで誤報率が上がるケースがある。
学習データに劣化初期のパターンが混入するためと考えられる。

---

## 現場への示唆：個体別＋共通モデルの併用

今回の結果から、単一モデルで全ての個体に対応するのは難しいことがわかった。

**提案する設計：**

```
個体別モデル（かかりつけ医）
→ この設備の正常を知っている
→ 誤報が少ない（誤報率約5%）
→ 徐々に劣化するタイプに強い

共通モデル（専門医）
→ 多くの個体のデータを持っている
→ 突然壊れるタイプにも反応できる
→ ただし誤報が増えるリスクがある

併用すると
→ 両方が「異常」→ 高確度で異常
→ 個体別のみ「異常」→ 要注意（その個体に特有の変化）
→ 共通のみ「異常」→ 要注意（他と比べて挙動が違う）
→ 両方「正常」→ 正常
```

**学習期間の設計指針：**

```
10〜20%が最適帯
→ 共通モデルの検出力が高い
→ 誤報率も比較的安定

30%まで増やすと
→ 共通モデルの検出力が落ちる
→ 劣化初期のパターンが「正常」として学習されるため
```

現場への適用イメージ：

```
新品設置後の最初の10〜20%稼働時間
→ 自動学習期間（個体別モデル構築）
→ 同時に共通モデルも並行稼働
→ 以降は両モデルで監視
→ どちらかが異常を出したら通知レベルを変える
```

---

## 示せていること・示せていないこと

**示せていること**
- 個体別学習は誤報率の安定化に有効
- 共通モデルは「突然壊れるタイプ」の検出に有効
- 学習期間10〜20%が最適帯
- 併用アプローチが単一モデルより優れている可能性

**示せていないこと**
- 「突然壊れるタイプ」を事前に識別する方法
- 併用時の最適な閾値設計（現場ごとに調整が必要）
- 実機データでの再現性（次のステップ）
